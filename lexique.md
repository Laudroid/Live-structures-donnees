**Algorithme** : Séquence d'instructions conçue pour résoudre un problème, caractérisée par une structure générale comprenant l'entrée, le traitement et la sortie. Peut être simple ou classique (recherche, tri).

**Algorithmique** : La démarche de compréhension et de formulation d'un problème en vue de sa résolution par un algorithme ; la matière étudiée qui prépare à la programmation.

**Affectation** : Opération qui consiste à attribuer une valeur à une variable, dont la syntaxe est définie en pseudo-code.

**Algorithmes de recherche simple** : Mécanismes pour trouver des éléments spécifiques au sein d'une structure de données, par exemple dans un tableau.

**Booléen** : Type de base représentant une valeur de vérité (vrai ou faux).

**Boucle POUR** : Structure itérative utilisée pour des répétitions définies, où le nombre d'itérations est généralement connu à l'avance.

**Boucle REPETER ... JUSQU’A** : Structure itérative qui répète un bloc d'instructions jusqu'à ce qu'une condition devienne vraie (testée en fin de boucle, garantissant au moins une exécution).

**Boucle TANT QUE** : Structure itérative qui répète un bloc d'instructions tant qu'une condition reste vraie (testée en début de boucle).

**Caractère** : Type de base représentant un seul symbole ou lettre.

**Chaînes** : Structures de données de base mentionnées comme manipulables.

**Constantes** : Éléments utilisés pour stocker des valeurs fixes qui ne changent pas pendant l'exécution d'un algorithme.

**Démarche algorithmique** : Les étapes à suivre pour concevoir un algorithme efficace.

**Données d’entrée** : Les informations ou valeurs nécessaires à un algorithme pour son exécution.

**Données de sortie** : Les résultats ou informations produites par un algorithme.

**Entier** : Type de base représentant les nombres entiers.

**Entrées / sorties** : Principes des opérations de lecture de données (entrées) et d'affichage de résultats (sorties).

**Expressions arithmétiques** : Combinaisons de valeurs, variables et opérateurs mathématiques, évaluées pour produire un résultat numérique.

**Expressions logiques** : Combinaisons de valeurs, variables et opérateurs logiques, évaluées pour produire un résultat booléen.

**Langage de programmation** : Outil permettant de traduire un algorithme en instructions exécutables par une machine (ex: C, Java, Python).

**Priorité des opérateurs** : Règles déterminant l'ordre dans lequel les opérations sont effectuées au sein d'une expression.

**Problème** : Ce qui doit être compris et formulé sous forme algorithmique.

**Programme** : La concrétisation d'un algorithme, préparant la transition vers un langage de programmation.

**Pseudo-code** : Langage informel utilisé pour décrire un algorithme et sa syntaxe (comme l'affectation ou les structures de contrôle).

**Réel** : Type de base représentant les nombres décimaux.

**SELON ... FAIRE** : Structure conditionnelle qui permet de choisir un bloc d'instructions à exécuter parmi plusieurs en fonction de la valeur d'une expression.

**SI ... ALORS** : Structure conditionnelle qui exécute un bloc d'instructions si une condition est vérifiée.

**SI ... ALORS ... SINON** : Structure conditionnelle qui exécute un premier bloc d'instructions si une condition est vérifiée, et un autre sinon.

**Structures conditionnelles** : Instructions (SI...ALORS, SI...ALORS...SINON, SELON...FAIRE) qui permettent l'exécution sélective de blocs de code selon des conditions.

**Structures de contrôle** : Éléments fondamentaux de l'algorithmique permettant de manipuler le déroulement d'un algorithme.

**Structures de données de base** : Moyens fondamentaux d'organiser les informations, incluant les tableaux et les chaînes.

**Structures itératives** : Instructions (POUR, TANT QUE, REPETER...JUSQU'A) permettant de répéter l'exécution d'un bloc de code.

**Structure générale d’un algorithme** : Présentation des différentes parties d'un algorithme, à savoir l'entrée, le traitement et la sortie.

**Tableaux** : Structures de données permettant de stocker une collection d'éléments, pouvant être unidimensionnels ou bidimensionnels.

**Tableaux bidimensionnels** : Tableaux organisés en lignes et colonnes, souvent utilisés pour représenter des matrices.

**Tableaux unidimensionnels** : Tableaux organisés sur une seule ligne ou colonne, représentant une liste d'éléments.

**Tris** : Principes d'organisation des éléments d'une collection dans un ordre donné.

**Types de base** : Catégories fondamentales de données, telles que entier, réel, booléen et caractère.

**Variables** : Éléments utilisés pour stocker des valeurs qui peuvent varier au cours de l'exécution d'un algorithme.
